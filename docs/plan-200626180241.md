## Tujuan MVP

Library Go yang, saat di-load di `main.go`, expose handler `/graph`. Buka di browser ‚Üí render flowchart auto-generate dari arsitektur backend: module ‚Üí controller ‚Üí service ‚Üí repository, dengan panah call antar-layer. Klik node ‚Üí loncat ke source di editor (VSCode/Cursor).

## Scope v0.1 (default ‚Äî redirect kalau salah)

| Dimensi | v0.1 | Fase nanti |
|---|---|---|
| Run mode | **dev-live** (analyze source saat startup) | prod baked `graph.json` (`go:generate` + `go:embed`) |
| Framework | **net/http + gin** | echo, fiber, chi |
| Pattern | **modular MVC** (controller‚Üíservice‚Üírepo) | hexagonal (port/adapter), clean arch |
| Call graph algo | **`static`/CHA** | RTA/pointer (resolve interface dispatch) |
| Editor jump | **vscode + cursor** (URL scheme) | JetBrains (REST API) |

## Langkah build

1. **Scaffold** ‚Äî `go mod init`, layout package analyzer/route/classify/graph/build/web + archview.go (public API).
2. **Analyzer** ‚Äî `go/packages` + `go/callgraph` (CHA). Tiap func node simpan pkg, name, recv, file:line:col.
3. **Route adapter** ‚Äî `RouteExtractor` interface, impl per framework (gin, net/http).
4. **Classify** ‚Äî heuristik folder/naming + config opsional.
5. **Emit graph** ‚Äî `graph.json`: nodes + edges + editorURL.
6. **Web** ‚Äî handler `/graph`, embed assets, render SVG, klik node ‚Üí editor.
7. **Example app** ‚Äî proof end-to-end.
