# Plan — archview-net (.NET port)

> Dibuat: 2026-06-22 13:57 WIB · Status: PLAN ONLY (belum dibangun)
> Repo target: **`github.com/eularixs/archview-net`** (terpisah dari `eularixs/archview`)
> Docs: **tetap 1** di `eularixs/archview-docs` (Go + .NET dalam satu situs)

## Tujuan

Port konsep archview ke .NET: library C# yang menganalisa backend .NET secara
statik dan me-render flow graph yang **sama** (endpoint → controller → service →
repository) di `/graph`. UX, skema graph, dan UI identik — yang beda cuma mesin
analisa (bahasa).

## Insight kunci: yang di-share vs yang beda

| Bagian | archview (Go) | archview-net (.NET) | Share? |
|---|---|---|---|
| Static analysis | `go/packages` + SSA + CHA | **Roslyn** (`Microsoft.CodeAnalysis`) | beda |
| Call graph | `go/callgraph` | Roslyn semantic model + `SymbolFinder` | beda |
| Route extractor | gin/echo/grpc/... | ASP.NET Core / Minimal API / gRPC / ... | beda |
| Layer classify | folder/naming + auto-layer | namespace/folder + auto-layer (algoritma SAMA) | logika sama |
| Ports / buses | interface + mediator | interface + **MediatR** | konsep sama |
| **graph.json schema** | nodes/edges/violation | **identik** | ✅ SHARE |
| **Web UI `/graph`** | embedded `index.html` (SVG) | **embedded `index.html` yang sama** | ✅ SHARE |
| Docs | archview-docs | archview-docs (section .NET) | ✅ SHARE |

**Yang bikin "docs tetap 1"**: graph schema + UI + konsep (auto-layer, ports,
buses, linter, click-to-source) identik lintas bahasa. Cuma halaman *frameworks*
+ *usage* yang split per-bahasa.

## Stack archview-net

- **Bahasa**: C# / .NET 8+.
- **Analisa**: Roslyn — `MSBuildWorkspace` load `.csproj`/`.sln`, `SemanticModel`
  per file, resolve `InvocationExpression` → `IMethodSymbol` buat call edge.
  Interface dispatch: `SymbolFinder.FindImplementationsAsync` (ekuivalen CHA).
- **Serve**: ASP.NET Core middleware — `app.UseArchview("/graph")` (extension
  method), atau host Kestrel mini. Handler net-agnostic.
- **Distribusi**: NuGet `Eularix.Archview` (atau `Archview`).
- **Run mode**: dev-live (analisa source `.csproj` saat startup). Butuh source +
  SDK di runtime (caveat sama kayak Go).

## Entry detection (.NET frameworks)

| Framework | Pola |
|---|---|
| **Minimal API** | `app.MapGet("/users", handler)`, `MapPost/Put/...`, `MapGroup("/api")` (prefix) |
| **MVC Controller** | atribut `[HttpGet("/users")]` / `[Route]` di method `ControllerBase`, `[ApiController]` |
| **gRPC** | service class turunan `XxxBase` (proto-gen) + `MapGrpcService<T>()` |
| **SignalR** | `Hub` subclass (realtime — analog WebSocket), `MapHub<T>()` |
| **FastEndpoints** | `Endpoint<TReq>` + `Get("/route")` di `Configure()` |
| **Carter** | `CarterModule` + `app.MapGet` |

## Patterns (.NET)

- Layered: namespace `Api`/`Controllers` → controller; `Application`/`Services`/
  `UseCases` → service; `Infrastructure`/`Repositories`/`Persistence`/`EF` →
  repository. + auto-layer (chain-based, algoritma sama).
- **CQRS via MediatR** (analog bus detection Go): deteksi `IRequestHandler<TCmd,
  TRes>` impls + `mediator.Send(cmd)` → tarik edge command→handler dari tipe
  request. MediatR = "the bus" di dunia .NET.
- Hexagonal: outbound port = interface di-implement adapter (EF Core
  `DbContext` repository) → ShowPorts.
- EF Core repository = outbound (DB).

## Skema graph (kontrak lintas-bahasa) — di-formalkan

Definisikan `graph.json` sebagai kontrak resmi (dokumentasikan di archview-docs):
```
Node  { id, kind(endpoint|func|port), label, layer, module, pkg, func, file, line, editorURL, method, path }
Edge  { from, to, kind(route|call|implements|dispatch), violation? }
```
Go + .NET dua-duanya emit ini. UI konsumsi ini. Bikin shared spec page.

## Fase build (nanti, saat greenlight)

- **F0 Scaffold**: repo `eularixs/archview-net`, struktur project C#, NuGet meta,
  port graph model + embed `index.html` (copy dari archview Go), middleware
  `/graph` + `/graph/data`. MIT, identity Eularix.
- **F1 Core analysis**: MSBuildWorkspace load + call graph (symbol-based) +
  classify + **auto-layer** (port BFS chain inference). Output graph.json.
- **F2 Routes**: Minimal API (`MapGet`) + MVC controller (`[HttpGet]`).
- **F3 Depth**: MediatR (CQRS) + EF Core repository (ports) + gRPC.
- **F4 Reach**: SignalR hub (realtime), FastEndpoints, Carter.
- **F5 Polish**: linter (reverse/skip/cross), skip generated (`.g.cs` /
  `// <auto-generated>`), click-to-source (vscode/rider/vs scheme).

## Open decisions (sebelum build)

- Nama NuGet: `Eularix.Archview` vs `Archview`.
- UI sharing: copy `index.html` ke archview-net, atau publish sebagai shared
  asset/submodule biar 1 source-of-truth. (Saran: copy dulu, formalkan schema,
  nanti pertimbangkan shared package.)
- Target .NET minimum (8 LTS?).
- MSBuildWorkspace butuh MSBuild di runtime — caveat deployment.

## Verdict feasibility

**Possible.** Roslyn = padanan kuat go/packages+SSA (semantic model + symbol
graph + FindImplementations). Pola entry .NET (Minimal API/MVC/gRPC/MediatR)
sama jelasnya buat di-extract. UI + schema + konsep di-reuse → effort fokus di
analisa Roslyn, bukan bikin ulang dari nol. Docs tetap 1 karena konsepnya sama.
