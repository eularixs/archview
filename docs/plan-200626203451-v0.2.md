# Plan ‚Äî archview v0.2

> Dibuat: 2026-06-20 20:34 WIB
> Basis: fitur tambahan yang dibahas (Tier 1/2/3) + web docs (fumadocs)
> v0.1 status: shipped ‚Üí `eularixs/archview` @ `v0.0.1-beta`

## Tujuan v0.2

Naikin archview dari "viewer" jadi **architecture guardrail + dev tool yang enak**, plus punya **web documentation** resmi (fumadocs). Fokus: lengkapin visi awal (outbound kebaca) + value unik (layer-violation) + UX matang.

## Workstream

### A. Analysis depth (Tier 1 ‚Äî value tertinggi)

1. **Outbound / external dependency detection** ‚≠ê
   - Deteksi repo/service nyentuh resource luar: `database/sql`, `pgx`, `gorm`, HTTP client (`net/http` client, `resty`), `redis`, kafka/amqp, gRPC client.
   - Render node external di kanan repo: `[postgres]`, `[http: stripe]`, `[redis]`.
   - Edge baru kind `outbound`. Node kind `external`.
   - Cara: scan call graph leaf yang masuk ke package dep tertentu (match import path / func signature). Heuristik per-driver.

2. **Layer-violation / arch-smell detector** ‚≠ê
   - Cek transisi edge salah: controller‚Üírepository (skip service), repository‚Üíservice/controller (arah kebalik), cross-module call.
   - Flag merah di edge + panel list "violations".
   - Cara: udah punya layer per node + edge; tinggal rule check di `build/`. Output field `violation` di Edge.

3. **Join prefix path**
   - `r.Group("/api")` + `.GET("/users")` = `/api/users`.
   - Track receiver var ‚Üí group prefix (gin `RouterGroup.Group`, net/http `http.StripPrefix`/sub-mux).
   - Cara: di route adapter, resolve chain `Group(...)` prefix per receiver.

### B. Web UX (Tier 2 ‚Äî murah, dampak besar)

4. **Search + filter** ‚Äî cari node by name; toggle per layer / per module (show/hide).
5. **Klik endpoint ‚Üí highlight full downstream path** sampai repo/external (BFS edge).
6. **Persist layout** ‚Äî `laneOffset` + `nodeDelta` + collapsed ke `localStorage` (per module path). Drag ga ilang pas refresh.
7. **Live reload** ‚Äî re-analyze on-request (opsi) atau fsnotify watch ‚Üí SSE refresh browser.

### C. Breadth (Tier 3)

8. **echo / fiber / chi** route adapter (interface udah siap).
9. **Prod-baked** ‚Äî `archview gen` CLI / `go:generate` ‚Üí emit `graph.json`, `go:embed`. Jalan di binary deployed (tanpa source runtime).
10. **Export** ‚Äî PNG / SVG / mermaid / dot dari graph.

### D. Web documentation (fumadocs) ‚Äî workstream paralel

11. **Fumadocs site** di `website/` (Next.js + fumadocs, bun).
    - Pages: Introduction, Quickstart, Installation, Configuration (Options), How it works, Layer classification, Frameworks, Patterns (MVC/hexagonal), UI guide, Examples, Limitations, Roadmap, Changelog.
    - Deploy Vercel (eularixs). Domain nyusul.

## Urutan saran

1. **D (fumadocs)** ‚Äî sekarang, paralel, ga ganggu Go code.
2. **A1 outbound + A3 path** ‚Äî paling nendang, lengkapin visi.
3. **A2 layer-violation** ‚Äî killer feature.
4. **B (UX)** ‚Äî search/filter, persist, highlight path.
5. **C** ‚Äî echo/fiber, prod-baked, export.

## Out of scope v0.2
- Runtime tracing (hit count per edge) ‚Äî beda arsitektur, fase 3.
- JetBrains editor adapter.
- Pointer analysis (ganti CHA) ‚Äî kecuali multi-impl jadi masalah nyata.

## Definition of done v0.2
- Outbound node kebaca di example (tambah repo yang query DB beneran).
- Violation ke-flag di contoh sintetis (controller‚Üírepo skip).
- Path benar (`/api/users`).
- fumadocs site build + deploy, isi lengkap, link dari README.
